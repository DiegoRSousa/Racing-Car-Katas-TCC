# Racing Katas
Neste repositório, você encontrará o código inicial para cinco problemas distintos. Eles podem ser o código que herdou de um código-base legado. Agora você quer escrever testes de unidade para eles, e isso é mais difícil do que precisa ser. Todos os fragmentos de código não seguem um ou mais dos princípios SOLID.

Para cada exercício, você deve identificar quais princípios SOLID não estão sendo seguidos pelo código. Há apenas uma classe que você está interessado em escrever testes neste momento. Como primeiro passo, tente obter algum tipo de teste antes de modificar a classe. Se os testes são difíceis de escrever é porque os princípios SOLID não estão sendo segudos.

Quando você tiver algum tipo de teste para se apoiar, refatore o código e torne o testável.  Tenha cuidado ao refatorar para não alterar a funcionalidade ou as interfaces nas quais outro código de cliente usa. (Imagine que há um código de cliente em outro repositório que você não pode ver agora). Adicione mais testes para cobrir a funcionalidade da classe especifica que você foi solicitado a testar.

Aplique o estilo de teste de unidade e o framework com o qual você está mais confortável. Você pode optar por usar stubs, mocks ou nenhum. Se você fizer isso, você pode usar a ferramenta que preferir.

1. **Exercício do sistema de monitoramento de pressão:** escreva testes de unidade para a classe Alarme. A classe Alarme foi projetada para monitorar a pressão do pneus e definir um alarme se a pressão sair forma do alcance esperado. A classe Sensor fornecida para o exercício simula o comportamento de um sensor de pneu real, fornecendo valores aleatórios mas realistas.
2. **Exerício Conversor de arquivo unicode para html:** escreva os testes de unidade para a classe ConversorArquivoUnicodeFileParaHtml. A classe ConversorArquivoUnicodeFileParaHtml foi projetada para formatar um arquivo de texto simples para exibição em um navegador. Para as versões Python e Java, existe uma classe adicional ConversorPaginasHtml, que é um pouco dificil de testar. Ela não só converte texto em um arquivo HTML, mas também é compatível com a paginação. Usada como um exercício de acompanhamento.
3. **Exercício distribuidor de bilhetes:** escreva testes de unidade para o DistribuidorDeBilhetes. A classe DistribuidorDeBilhetes foi  projetada para ser usada para gerenciar um sistema de filas em uma loja. Pode haver mais de um distribuidor de bilhete, mas o mesmo bilhete não deve ser emitido para clientes diferentes.
4. **Exercício Sistema de telemetria:** escreva os testes de unidade para a classe ControleDeDiagnosticoDeTelemetria. A responsabilidade da classe ControleDeDiagnosticoDeTelemetria é estabelecer uma conexão com o servidor de telemetria (através do ClienteTelemetria), enviar uma solicitação de diagnóstico e receber com sucesso a resposta que contém as informações de diagnóstico. A classe ClienteTelemetria fornecida para o exercício simula o comportamento real de ClienteTelemetria e pode responder com as informações de diagnóstico ou uma sequência aleatória. A classe ClienteTelemetria se conectaria e se comunicaria com o servidor de telemetria via tcp/ip.
5. **Exercício Entre os Melhores:** (note que este exercício ainda está sendo desenvolvido) Escreva testes de unidade para a classe EntreOsMelhores, incluindo corridas com automóveis autodirigidos. O EntreOsMelhores calcula os pontos dos motoristas e os rankings com base em resultados de várias corridas.
